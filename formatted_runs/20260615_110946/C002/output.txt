Technical Narrative:
The LoanApplicationService class is marked with @Service, making it part of
the business logic layer.

source_line_ranges: ["f02_1-f02_420"]

The method createApplication first calls validate(request), which checks
whether loan amount is less than or equal to zero. If true, an
IllegalArgumentException is thrown and execution stops immediately.

source_line_ranges: ["f02_1-f02_420"]

If validation passes, customerRepository.findById is called to retrieve
customer information, transferring control to C003.

source_line_ranges: ["f02_1-f02_420"]

The LoanRequest is converted into a LoanEntity using loanMapper.toEntity, then
its status is set to "PENDING".

source_line_ranges: ["f02_1-f02_420"]

loanRepository.save persists the entity, transferring control to C004 for
database interaction.

source_line_ranges: ["f02_1-f02_420"]

Finally loanMapper.toResponse converts the entity into a LoanResponse returned
to the controller.
