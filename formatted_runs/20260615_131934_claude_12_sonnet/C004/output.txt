business_functional_name: |
Loan Application Record Structure And Lifecycle Tracking

core_business_functionality: |
This chunk defines the persistent record structure for a loan application,
capturing its applicant, financial details, current status, decision outcome,
and key timestamps. It also automatically tracks when the record was last
modified and provides a way to determine whether the requested amount
qualifies as a high-value request.

business_function: |
Each loan application record stores the applicant it belongs to.
Each loan application record stores the amount requested and, if applicable,
the amount approved.
Each loan application record stores its purpose, term, and current status.
Each loan application record stores a rejection reason if it was rejected.
Each loan application record stores the applicant's credit score at the time
of submission.
Each loan application record stores when it was submitted, when it was
decided, and when it was last modified.
A loan application record can determine whether its requested amount exceeds a
given threshold.
