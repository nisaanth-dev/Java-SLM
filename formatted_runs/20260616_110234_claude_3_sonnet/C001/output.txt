business_functional_name: |
Loan Application Record Definition

core_business_functionality: |
This component defines the central record that captures all details of a
borrower's request for a loan, from the moment of submission through final
credit decision. It governs how loan application data is stored, what fields
are mandatory, how the application's lifecycle state is tracked, and what
lifecycle events trigger automatic timestamp and status initialization.

Store the repayment duration in months chosen by the borrower.
Maintain the current status of the application throughout its lifecycle.
Classify the application by loan product type.
Capture the borrower's stated purpose for the loan.
Record the borrower's credit score at the time of assessment.
Track the borrower's debt-to-income ratio used in credit decisioning.

Record internal underwriter commentary on the application.
Track the date and time the application was formally submitted.

Automatically set creation and last-updated timestamps on the application
record.
Initialize a new application to the Draft status if no status is supplied at
creation time.
Ensure the document collection is always initialized when a new application is
first saved.
Increment the record version and refresh the last-updated timestamp whenever
the application is modified.
Allow documents to be attached to an application while maintaining the

Allow documents to be detached from an application while maintaining the
bidirectional relationship.
Determine whether an application is in a state that allows editing by the
