Imports
This chunk imports Spring Framework configuration and client annotations
(`Bean`, `Configuration`) which provide the capabilities to register
manageable components in the application context. It imports `RestTemplate`,
`RestTemplateBuilder`, and `ClientHttpRequestInterceptor` from
`org.springframework.web.client` and `org.springframework.http.client` to
construct and customize HTTP communication objects. It imports
`CollectionUtils` from `org.springframework.util` for null-safe list checks,
and standard Java `Duration`, `ArrayList`, and `List`.

Class-Level Declarations
The `IntegrationConfig` class is annotated with `@Configuration`, meaning it
declares one or more `@Bean` methods and may be processed by the Spring
container to generate bean definitions and service requests at runtime.

Methods
The `externalServiceClient` method returns a `RestTemplate` and is annotated
with `@Bean`, meaning its return value is registered as a managed component.
It accepts a `RestTemplateBuilder` named `builder`.

It declares a local variable `restTemplate` of type `RestTemplate`. It assigns
this variable the result of a chain of builder calls:
`.setConnectTimeout(Duration.ofSeconds(5))` setting a five-second connection
limit, `.setReadTimeout(Duration.ofSeconds(15))` setting a fifteen-second read
limit, and finally `.build()`.

It declares a `List<ClientHttpRequestInterceptor>` named `interceptors` and
assigns it the result of `restTemplate.getInterceptors()`. An `if` statement
checks whether `CollectionUtils.isEmpty(interceptors)` is true. If this
condition is true, it assigns a `new ArrayList<>()` to `interceptors`. If
false, execution continues.

It calls `interceptors.add()` with a lambda expression representing the
interceptor logic. For every outbound request, the lambda accesses the request
headers via `request.getHeaders().add("X-System-Auth",
"ACME_CLAIMS_INTERNAL")` to inject a security token, and then returns the
result of `execution.execute(request, body)` to proceed with the call.

Finally, it calls `restTemplate.setInterceptors(interceptors)` to apply the
modified list, and returns the fully configured `restTemplate`.
