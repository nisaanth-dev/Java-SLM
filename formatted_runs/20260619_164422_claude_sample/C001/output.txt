business_functional_name: |
Loan Application Submission And Status Management

core_business_functionality: |
This component exposes the primary touchpoints through which external clients
submit new loan applications, retrieve application details, and manage
application status transitions throughout the loan lifecycle. It acts as the
authoritative gateway for all inbound loan origination activity, enforcing
that only well-formed requests progress into the organisation's processing
pipeline.

business_function: |
A new loan application is accepted from an external client and forwarded into
the processing pipeline, returning a unique reference upon successful
creation.
A previously submitted loan application is retrieved in full by its assigned
reference number, enabling downstream consumers to inspect all recorded
details.
All loan applications associated with a specific applicant are retrieved as a
collection, or a no-content signal is issued when none exist.
The recorded status of an existing loan application is advanced to a new
value, along with optional remarks, finalising that stage of the underwriting
workflow.
The current status of a loan application is retrieved in isolation, supporting
lightweight polling by front-end or downstream systems without fetching full
application data.
